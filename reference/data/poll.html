Poll Model

Opinion question during an event

JSON

  {
   "question": {
    "type": "Poll",
    "id": 1,
    "tagline": "Just for fun.",
    "commentary": "",
    "choices": [
     {
      "submit": "a",
      "display": "Yes"
     },
     {
      "submit": "b",
      "display": "No way"
     }
    ],
    "body": "Will we see a stranger force out than that today?",
    "urls": [
     {
      "answer": "/polls/1/selections",
      "results": "/polls/1"
     }
    ]
   }
  }

Available via

Live Event check-in API calls in the preroll array. Multiple Choice Prompt WebSockets event.

Questions

None.

Summary

Data Usage

id
Used for unique identification of the chat message. Currently not be used.
type
The literal string "Poll"-- this indicates you should expect Poll results.
body
Body of the question, should receive primary display.
commentary
Commentary on the question, can receive italicized or smaller display to taste.
tagline
Description of question type, usually indicates whether question will be scored.

Choices

submit
If the user selects this choice, send this answer when Answering a multiple-choice question.
display
Use this text to display to the user.

URLs

answer
Submit to this path to answer this question.
results
Issue a GET request to this path to get an answer breakdown
