Trivia Model

Question about the past, during an event

JSON

{
   "question": {
    "type": "trivia",
    "id": 1,
    "tagline": "Correct answer worth 10 points.",
    "commentary": "13 SEC Titles, too.",
    "choices": [
     {
      "submit": "a",
      "display": "The University of Alabama"
     },
     {
      "submit": "b",
      "display": "Texas A&M University"
     },
     {
      "submit": "c",
      "display": "University of Kentucky"
     }
    ],
    "body": "At which school did Paul 'Bear' Bryant rack up 232 wins and 6 National Titles?",
    "urls": [
     {
      "answer": "/trivias/1/selections",
      "results": "/trivias/1"
     }
    ],
    "scored": false
   }
  }

Available via

Live Event check-in API calls in the preroll array. Multiple Choice Prompt WebSockets event.

Questions

How could/should we indicate which answer is correct? In the choices array, higher up, or somewhere else?

Summary

Data Usage

id
Used for unique identification of the chat message. Currently not be used.
type
The literal string "Trivia"-- this indicates you should expect trivia results.
body
Body of the question, should receive primary display.
commentary
Commentary on the question, can receive italicized or smaller display to taste.
tagline
Description of question type, usually indicates whether question will be scored.

Choices

submit
If the user selects this choice, send this answer when Answering a multiple-choice question.
display
Use this text to display to the user.

URLs

answer
Submit to this path to answer this question.
results
Issue a GET request to this path to get an answer breakdown
