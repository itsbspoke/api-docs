Membership Model

A record of group inclusion

JSON

{
 "membership": {
  "status": "pending",
  "id": 668,
  "group": {
   "id": 131,
   "name": "Test Group"
  },
  "user": {
   "avatar_url": null,
   "id": 2,
   "name": "@chrishalligan",
   "level": 1
  }
 }
}

Available via

Invite member to group, Group Listing API calls. User notification WebSockets event.

Questions

None.

Summary

Data Usage

id
Used for unique identification of the membership. Required to accept or ignore an invitation.
status
Literal strings: accepted, owned, ignored, pending. Ignored invitations should not be displayed. Pending invitations should be presented for action. Active and owned memberships should be reflected in gameplay.

References

The User summary and Group summary form are contained within this model.
